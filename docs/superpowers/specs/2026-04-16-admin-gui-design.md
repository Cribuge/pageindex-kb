# Admin 管理界面设计方案

## 概述

将知识库系统的管理功能（文档管理、上传、系统设置）从主 web UI 中分离出来，创建一个独立的 `admin.html` 管理后台。主 web `index.html` 精简为只保留智能对话和文档查看功能。

## 架构

```
web/
  index.html     # 精简版：对话 + 文档查看
  admin.html     # 新建：文档管理 + 上传 + 系统设置
  nginx.conf     # 新增 /admin/ 路由
```

- 两个 HTML 完全独立，通过 nginx 区分路径
- 后端 API 不变，所有管理功能通过 API 调用
- 沿用现有蓝白配色方案（深蓝侧边栏 + 浅蓝灰内容区）

## 功能模块详情

### 1. 文档管理

**分类侧边栏**
- 固定宽度 180px，内部垂直滚动
- 分类项：全部(n) / 各分类名(n) / 未分类(n)
- 点击分类切换右侧列表，同时高亮当前分类
- "全部文档"始终在顶部

**文档列表**
- 网格布局，与现有 `index.html` 中的文档列表一致
- 列：checkbox / 文档名称 / 格式 / 大小 / 上传日期 / 操作
- 操作按钮：重命名（铅笔图标）、分类（文件夹图标）、树索引（图表图标）、删除（垃圾桶图标）
- 支持全选 checkbox
- 搜索框：实时过滤文件名
- 刷新按钮：重新加载列表

**批量操作栏**
- 选中文档后显示：已选择 N 份 | 批量删除 | 批量重命名 | 批量设置分类 | 取消选择
- 批量删除：弹出确认框，调用 `POST /documents/batch-delete`
- 批量重命名：prompt 输入新名称前缀/后缀或直接替换
- 批量设置分类：弹窗选择目标分类

### 2. 上传文档

**上传区**
- 拖拽上传区（虚线边框，hover 高亮）
- 点击触发文件选择（accept 限制允许的类型）
- 显示允许的文件类型提示

**上传队列**
- 实时列表显示每个文件的上传状态
- 状态：等待中 / 上传中 / 已加入队列（绿色） / 失败（红色+错误原因）
- 错误提示与 `index.html` 一致（413 文件过大、400 格式不支持、500 服务器错误）
- 上传完成后自动刷新文档列表

### 3. 系统设置

**模型配置卡片**
- LLM 模型：select 下拉，从 Ollama API 获取可用模型列表
- 温度：range slider 0-2，步进 0.1，显示当前值
- 最大生成长度：number input，范围 256-8192
- 系统提示词：textarea，placeholder 显示默认值

**检索配置卡片**
- TOP_K：number input，范围 1-20
- 搜索深度：number input，范围 1-10

**树索引配置卡片**
- 树最大深度：number input
- 最大子节点数：number input
- 上下文字符阈值：number input

**操作**
- 顶部"保存设置"按钮 → `PUT /config`
- 底部提示："参数变更仅对新上传或重新处理的文档生效"

## 设计规范

### 配色
```css
--bg: #0d2a5f          /* 侧边栏背景 */
--bg-deep: #eef2f8      /* 内容区背景 */
--surface: #ffffff      /* 卡片背景 */
--surface-hover: #e8f2ff
--border: #c2d9f8
--text: #1a2a3a
--text-secondary: #4a6a8a
--accent: #1a5fc8
--accent-dim: rgba(26,95,200,0.1)
--green: #16a34a
--red: #dc2626
--amber: #d97706
```

### 侧边栏
- 宽度：200px
- 背景：`--bg`（深蓝）
- 文字：白色，字号 13px
- hover：背景 `rgba(255,255,255,0.12)`
- active：背景 `rgba(255,255,255,0.2)` + 左侧 3px 白色竖条

### 内容区
- 背景：`--bg-deep`
- 卡片：白色背景，圆角 12px，1px 边框
- 间距：内边距 20px，卡片间距 16px

### 按钮
- 主按钮：背景 `--accent`，hover `#1447b0`，文字白色
- 危险按钮：背景 `--red`
- 次要按钮：背景透明，背景 hover `var(--surface-hover)`

## 路由配置（nginx.conf）

```nginx
location /admin/ {
    alias /usr/share/nginx/html/admin/;
    try_files $uri $uri/ /admin.html;
}

location / {
    try_files $uri $uri/ /index.html;
}
```

## 实施步骤

1. 创建 `web/admin.html` 骨架（HTML + CSS 变量）
2. 实现侧边栏导航 + 切换逻辑
3. 实现文档管理面板（分类列表 + 文档网格 + 批量操作）
4. 实现上传文档面板（拖拽 + 队列）
5. 实现系统设置面板（3 张卡片 + 保存）
6. 精简 `web/index.html`（移除上传和设置面板）
7. 更新 `web/nginx.conf` 添加 `/admin/` 路由
8. 测试完整流程

## 验证方式

1. `docker compose build web && docker compose up -d web`
2. 打开 http://localhost:3001 — 对话界面正常
3. 打开 http://localhost:3001/admin/ — 管理界面正常
4. 管理界面上传文档 → 文档出现在文档管理列表
5. 修改系统设置 → 保存 → 刷新页面 → 值保持
6. 批量选择文档 → 批量删除 → 验证删除成功
